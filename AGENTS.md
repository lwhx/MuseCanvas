## 技术栈（严格限制）

前端：

- Vue 3（使用 Composition API + `<script setup>`）
- TypeScript（严格模式）
- Vite
- shadcn-vue / Reka UI（UI 组件库）
- Tailwind CSS（样式方案）
- Pinia（状态管理）
- Vue Router

后端：

- TypeScript
- Next.js（API Routes）
- PostgreSQL
- Redis
- Docker Compose
- Nginx 反向代理

**禁止引入上述技术栈之外的框架或库**（工具类小依赖如 lodash-es、date-fns 等需先确认）。

## 项目约定

### 1. 安全红线（不可违反）

- **禁止**提交任何密钥、密码、Token 到代码仓库
- **禁止**将 API Key、数据库密码等放在前端代码中
- **禁止**跳过鉴权直接暴露接口
- **禁止**用 mock 数据假装功能已完成
- 必须使用 `.env.example` 管理环境变量模板

### 2. 数据库规范

- 主键使用 UUID
- 时间字段使用带时区的时间戳
- 金额/额度类字段避免浮点数，使用整数或 Decimal
- 使用软删除（`deleted_at`）而非物理删除
- 核心表应包含：`tenant_id`（多租户场景）、`created_by`、`created_at`、`updated_at`

### 3. API 接口规范

统一响应格式：

```json
// 成功
{ "success": true, "data": {} }
// 失败
{ "success": false, "error": { "code": "ERROR_CODE", "message": "说明" } }
```

- 禁止暴露底层数据库异常、第三方 API 错误信息给前端
- 所有输入需校验，所有输出需过滤

### 4. 多租户隔离（如适用）

- 数据查询必须带 `tenant_id` 条件
- 权限校验在服务端完成，不能依赖前端隐藏 UI
- 禁止跨租户访问资源

### 5. 缓存与异步

- Redis 等缓存层仅用于非核心数据（队列状态、限流、临时缓存）
- 核心业务数据必须持久化到数据库
- 异步任务（如 AI 生图）不依赖前端长连接

### 6. 前端原则

- **必须使用** Vue 3 Composition API + `<script setup>` 语法
- 组件优先使用 **shadcn-vue / Reka UI**
- 样式使用 **Tailwind CSS**，禁止引入其他 CSS 框架（如 Element Plus、Ant Design Vue）
- 状态管理使用 **Pinia**，禁止引入 Redux、MobX 等
- 路由使用 **Vue Router**
- **禁止引入技术栈列表之外的 UI 库或状态管理库**
- 避免引入不必要的新依赖，新增依赖需说明理由

## Agent 工作流

### 修改前检查清单

开始修改代码前，确认：

1. [ ] 理解当前修改涉及的模块边界
2. [ ] 是否涉及鉴权/权限变更
3. [ ] 是否涉及数据模型变更（需 migration）
4. [ ] 是否涉及多租户隔离逻辑
5. [ ] 是否涉及第三方服务调用

### 完成后检查清单

每次修改完成后，验证：

1. [ ] 类型检查通过（`typecheck`）
2. [ ] Lint 通过
3. [ ] 相关测试通过
4. [ ] 接口鉴权正常
5. [ ] 无敏感信息泄露
6. [ ] 不破坏现有功能

### 功能文档完成门禁

当工作对应 `docs/{序号}-{功能名}/design.md` 时，功能文档是交付物的一部分：

1. 编码开始前应按 `docs/AGENTS.md` 创建或更新同目录 `impl.md`。
2. 达到本次功能的终止条件后，必须在最终回复用户前创建或更新同目录 `result.md`。
3. `result.md` 必须以实际代码和验证结果为准，记录完成项、变更摘要、验证命令及结果、与设计的偏差和遗留问题；不得把未执行的测试或未接通的外部服务写成已完成。
4. 最终回复中声称功能“完成”前，必须确认 `result.md` 已存在且反映最新实现。缺少 `result.md` 时，功能不视为完成。
5. 功能完成后的补丁、联调修复或验收结论发生变化时，必须同步更新 `result.md`，不能只修改代码或 `design.md`。

`docs/AGENTS.md` 对文档结构和生命周期有更具体的约束，处理 `/docs` 下的功能时必须同时遵循。

### 禁止事项

- 不得为"快速完成"而跳过权限校验
- 不得硬编码用户 ID 或租户 ID
- 不得绕过业务校验逻辑
- 不得删除已有代码而不说明原因
- 不得在未理解上下文的情况下大规模重构
